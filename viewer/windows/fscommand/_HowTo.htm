PipeBeama / Pipescript

Was ist PipeBeama ?
PipeBeama ist ein Zusatztool für Macromedia Flash-Projector-Files ab der Version 5 das 
den Funktionsumfang durch zusätzliche externe Befehle erweitert. Hierbei wird nicht wie
bei anderen Tools das Projektorfile modifiziert oder ein eigener Player verwendet.
Ein Standard-Projektorfile, das z.B. mit Macromedia Flash erstellt wurde, reicht hier völlig
aus. Macromedia hat aus Sicherheitsgründen das Actionscript-Kommando "fscommand"
stark eingeschränkt. Dadurch ergeben sich viele Nachteile bei der Produktion von
Multimediaanwendungen, wie z.B. interaktive Produkt- und oder Installations-CD`s.
PipeBeama fungiert sozusagen als Bindeglied zwischen Projektor-File und Betriebssystem
und stellt so dem Entwickler nützliche Funktionen zur Verfügung.

Welche Betriebssysteme werden unterstützt ?
PipeBeama ist zur Zeit nur für die Windows-Plattform erhältlich. Der Code wurde so angelegt,
daß PipeBeama unter den Betriebssystemen Win95, Win98, Me, NT, 2000 und XP lauffähig
sein sollte. Bugs, Fehlfunktionen, Anregungen, Tipps, Kritik etc. bitte an:
pipebeama@webmechaniker.de 

Brauche ich PipeBeama ?
Ob man PipeBeama benötigt oder nicht, muß man selber erkunden.
Jedenfalls sind die Möglichkeiten die dieses Tool bietet, bzw. in der Zukunft noch bieten wird, 
theoretisch "unendlich", da PipeBeama als C++ Anwendung natürlich alle Funktionalitäten 
einer höheren Programmiersprache zur Verfügung stellen kann.. 
Zur Zeit befindet sich PipeBeama im Alpha-Release, sodaß noch zahlreiche Befehle folgen
und auch noch evtl. Bugs behoben werden müssen. 
Sachdienliche Hinweise bitte an pipebeama@webmechaniker.de

Wie funktioniert PipeBeama ?
PipeBeama ist ein Socketserver, der auf Befehle (Pipescript) reagiert und so verschiedene 
Aufgaben für das Flash Projektor-File erledigen kann. Macromedia Flash Actionscript bietet dem
Entwickler eine sogenannte XML-Socket-Klasse, mit deren Methoden die Pipescriptbefehle
an PipeBeama gesendet werden. Diese werden dort verarbeitet und senden eine entsprechende 
Antwort an das Flash-Projektor-File zurück. So steht Ihnen eine duplexfähige Kommunikations-
schnittstelle zur Verfügung. Zum Beispiel werden bald auch Chatsysteme oder Netzwerkspiele
über das lokale Netzwerk möglich sein.

Wie integriert man PipeBeama in Flash-Projektor-Files ?
Die Integration von PipeBeama ist kinderleicht. Für einen einfachen Einstieg empfiehlt es sich das
Flash-Demofile, daß sich in dem Ordner "FLA" befindet, durchzuarbeiten. Hinweis: In dem Demofile
wird PipeBeama zur Demonstration über eine Schaltfläche gestartet. Es empfiehlt sich aber in der
Praxis ein einfacher direkter Aufruf am Anfang des Flashfilms, damit PipeBeama genug Zeit hat sich
zu initialisieren.

Grundsätzlich muß PipeBeama einmal gestartet werden. 
Hierzu ist das Actionscript-Kommando fscommand ("exec","PipeBeama.exe"); zu verwenden.
Die Datei PipeBeama.exe muß sich ab der Flash Version 6 in dem Ordner "fscommand" befinden.
Flash 5 Anwender können den Ordner  noch angeben in dem sich die PipeBeama.exe befindet.
Liegt diese ebenfalls im fscommand Ordner so sieht die Syntax für Flash 5 wie folgt aus:
fscommand ("exec","fscommand/PipeBeama.exe");
Bevor PipeBeama einen Befehl entgegen nehmen kann, muß sich das Projektor-File einmalig anmelden, 
sozusagen "Hallo da bin ich" sagen. Hierzu erstellen Sie einfach ein eigenes Objekt der Klasse XMLSocket.
In dem Beispiel wurde der Name MyBroadcaster verwendet.

MyBroadcaster = new XMLSocket();
MyBroadcaster.connect("127.0.0.1",2000);

Die 127.0.0.1 kennzeichnet Ihren Computer und wird auch in der Computersprache als "localhost" bezeichnet.
Die 2000 ist die Portnummer, die zur Kommunikation mit PipeBeama verwendet wird. 

Nach erfolgreicher Anmeldung vergeben Sie einen Kommunikationsnamen, sodaß PipeBeama ihren Flash-
Film eindeutig zuordnen kann.

Befehl: <PB_connect><user></user></PB_connect>
Erläuterung: Verbindet einen Client mit PipeBeama
Parameter: <user></user> string
Rückgabe: <return></return> 1 erfolgreich -1 Fehler
  <message></message> OK
  user already exists Der User (Film) ist trotzdem angemeldet. Eine Fehlerbetrachtung wird dem Anwender überlassen.

Hierzu ein Actionscript Beispiel:

sSend = "<PB_connect>\n" +
                "<user>Testuser</user>\n" 
                +"</PB_connect>\n";

MyBroadcaster.send (sSend);

sSend ist eine einfache Variable vom Typ String. Die Stringverkettung mit den abschließenden Zeilenumbrüchen \n
wurde nur zur Demonstration gewählt. Natürlich wäre folgende Schreibweise auch möglich:
sSend = "<PB_connect><user>Testuser</user></PB_connect>";
Nun sind Sie fertig und können die nachfolgenden Befehle an PipeBeama senden und verarbeiten.
Diese werden mit der Methode "send" der Klasse XML, wie schon oben gesehen, versendet.
MyBroadcaster.send ("....");

Pipescript ist case-sensitive, also unbedingt auf die Groß- Kleinschreibung achten!

TIPP: Zur besseren Verarbeitung der Rückgabewerte sollte der Befehl System.useCodepage = true;
am Anfang des Films gesetzt werden. Dieser erlaubt den Zugriff auf den erweiterten Zeichensatz des
Betriebssystems.

Befehl: <PB_goodbye></PB_goodbye>
Erläuterung: Beendet PipeBeama.
PipeBeama arbeitet nach erfolgreicher Initialisierung als Hintergrundprozeß.
Auch wenn der Flashfilm beendet wurde, bleibt PipeBeama aktiv, wenn nicht zuvor dieser Befehl aufgerufen wurde. PipeBeama ist in der Programmiersprache C++ entwickelt worden und sehr Systemressourcensparend aufgebaut. Würde PipeBeama durch den Flashfilm nicht beendet werden, ist das folglich auch nicht sehr relevant. Die einzige Aufgabe besteht lediglich auf einen Port anfragen entgegen zu nehmen. Betrachtet man z.B. die Aufgaben einer softwaretechnischen Firewall etc. so muß diese innerhalb von Millisekunden "alle" Ports (das sind immerhin 65536) abhorchen und gleichzeitig noch sonstige Aufgaben erledigen !!!
Natürlich ist PipeBeama nach einem Rechnerneustart nicht mehr aktiv.
Aber vor allem bei der Entwicklung ist dieses Verhalten sehr angenehm, da PipeBeama nur einmal gestartet werden muß und so alle Befehle immer zur Verfügung gestellt werden können.
Mehrfaches (versehentliches) Starten von PipeBeama ist nicht möglich, da PipeBeama den neuen Prozess erkennt und diesen automatisch wieder beendet.
Parameter: keine
Rückgabe: keine

Befehl: <PB_open><dialog></dialog><title></title><ext></ext><maxfiles></maxfiles></PB_open>
Erläuterung: Öffnet einen File-Open- oder Path-Open-Dialog. 
Parameter: <dialog></dialog> singlefile Der Anwender kann eine Datei auswählen
<dialog></dialog> multifile Der Anwender kann mehrere Dateien selektieren
<dialog></dialog> path Der Anwender kann einen Pfad auswählen. Dieser kann dann z.B. mit dem Befehl <PB_getdirs>  weiter verarbeitet (ausgelesen)  werden.
<title></title> string Setzt den Titel des Dialoges
<ext></ext> Setzt einen Filter nach Wildcard-Regeln für die Dialogtypen singlefile oder multifile.
Beispiele:
*.* Alle Dateien stehen zur Auswahl
  *.pdf Dateien mit der Endung 'pdf' stehen zur Auswahl
  d*.mp3 Dateien die mit dem Buchstaben 'd' beginnen und die 
Endung 'mp3' haben stehen zur Auswahl
  <maxfiles></maxfiles> Ganzzahl (nur für multifile Dialog) 
Setzt die maximale Anzahl der Dateien die zurückgegeben werden sollen, obwohl der Anwender ein höhere Anzahl im multifile Dialog ausgewählt hat. Sollten Sie dem Anwender einen multifile Dialog anbieten, empfiehlt es sich diesen Filter auf eine sinnvolle Anzahl zu reduzieren, da der Flashplayer vor allem auf 'schwachen' System evtl. die Datenmengen nicht mehr verarbeiten kann.
Wird maxfiles leer gelassen, so können vom Anwender
'unendlich' (<=FreeRAM) viele Dateien ausgewählt werden.
Rückgabe: <return></return> 1 erfolgreich                   -1 Fehler
<message></message> string mit Rückgabewert bzw. erweiterten Informationen zur Ausführung des Befehls
